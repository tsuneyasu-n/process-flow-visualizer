# Process Flow Visualizer

業務プロセスを可視化し、AIでボトルネックを検出・改善提案を行うDX支援ツール

## 機能

- **フロー編集**: ドラッグ&ドロップで業務フローを作成
- **5種類のノードタイプ**: 開始、終了、作業、判断、待機
- **AI分析**: GPT-4o-miniによるボトルネック検出と改善提案
- **保存/読込**: ブラウザ内保存、JSONエクスポート/インポート
- **デモデータ**: 経費精算、採用プロセス、顧客問い合わせ対応の3フロー

## 技術スタック

- Next.js 14 (App Router)
- TypeScript
- React Flow (@xyflow/react)
- Tailwind CSS
- Zustand (状態管理)
- OpenAI API (GPT-4o-mini)

## セットアップ

### 1. 依存関係のインストール

```bash
cd process-flow-visualizer
npm install
```

### 2. 環境変数の設定

`.env.local` ファイルを作成:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

http://localhost:3000 でアクセス

## 使い方

### フローの作成

1. 左側のツールパネルからノードを追加
2. ノードをドラッグして位置を調整
3. ノードの下部をドラッグして別のノードに接続
4. ノードをクリックして詳細を編集

### AI分析

1. フローを作成
2. 右上の「AI分析」ボタンをクリック
3. ボトルネックと改善提案が表示される

### デモデータの読込

1. 左側の「デモデータを読込」ボタンをクリック
2. 経費精算/採用プロセス/顧客問い合わせ対応から選択

## ポートフォリオとしての見どころ

### DX企画職向けアピールポイント

1. **課題設定**: 属人化した業務フローの可視化という実務課題に対応
2. **AI活用**: 生成AIを業務改善に活用する実践例
3. **ユーザー体験**: ドラッグ&ドロップによる直感的な操作
4. **データ駆動**: 所要時間データに基づく客観的な分析

### 技術的なアピールポイント

- モダンなReact/Next.js開発
- 状態管理(Zustand)の適切な使用
- TypeScriptによる型安全な開発
- OpenAI APIの統合
